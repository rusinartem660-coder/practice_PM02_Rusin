# Тестовый фреймворк для сервиса валидации заказов

Данный репозиторий содержит спецификацию контракта, эталонную реализацию валидатора (заглушку) и комплексный набор тестов для проверки как эталонной, так и будущей реальной реализации ("черного ящика").

## Структура проекта
* `specification.md` — Спецификация контракта, включая JSON-схемы и Decision Table.
* `test_cases.md` — База тест-кейсов (30+ сценариев).
* `fake_validator.py` — Эталонный валидатор (Mock) с поддержкой симуляции задержек и режима хаоса (`chaos_mode`).
* `conftest.py` — Фикстуры инициализации валидатора для PyTest.
* `test_validate_order.py` — Автотесты (Параметризованные тесты, Property-Based тесты, временные границы).

## Как запустить тесты

### 1. Требования
Для работы тестов необходим Python версии 3.8+ и следующие библиотеки:

bash

pip install pytest hypothesis pydantic

### 2. Запуск тестов
Чтобы выполнить весь тестовый люкс:

bash

pytest -v test_validate_order.py

## Какие свойства проверяются
1. **Соответствие схеме (Спецификация как контракт):** неверные типы данных или пропущенные обязательные поля автоматически отклоняются.
2. **Бизнес-инварианты:**
   * Свойства нового пользователя (лимит 15k).
   * Границы времени продажи и проверки возраста для алкогольной продукции (включая критические секунды перехода границ 08:00 и 23:00).
   * Монотонность начисления штрафного скоринга рисков (любое ухудшение параметров увеличивает или оставляет скор равным, но никогда не уменьшает).
